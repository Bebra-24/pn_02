## 📖 О проекте

**«АгроКонтроль»** — это информационная система, разработанная в рамках дипломного проекта для автоматизации процессов управления производством средств защиты растений.

Система объединяет в единую цифровую среду всех участников производственного процесса:
- 👨‍🔬 **Технологов** — для управления рецептурами и производством
- 🧪 **Лаборантов** — для контроля качества продукции
- 🔧 **Аппаратчиков** — для выполнения технологических операций
- 👔 **Руководителей** — для анализа и принятия решений

### Основные возможности

| Модуль | Функции |
|--------|---------|
| **Модуль технолога** | Управление продукцией, рецептурами, технологическими картами, заказами, партиями, настройка экструдера, мониторинг, анализ отклонений, отчеты |
| **Модуль лаборатории** | Контроль качества сырья и готовой продукции, ввод результатов анализов, автоматическая проверка нормативов, формирование протоколов |
| **API сервер** | REST API с JWT-аутентификацией, 25 эндпоинтов, Swagger документация |

## 🎯 Актуальность

### Проблемы традиционного подхода

| Проблема | Последствие |
|----------|-------------|
| Ручной ввод данных в Excel | Ошибки, потеря времени |
| Разрозненные источники информации | Невозможность анализа |
| Отсутствие прослеживаемости партий | Трудно найти причину брака |
| Нет автоматического контроля качества | Риск выпуска некачественной продукции |
| Сложность управления версиями рецептур | Использование устаревших данных |

### Результат внедрения

| Показатель | До внедрения | После внедрения |
|------------|--------------|-----------------|
| Время на рецептуру | 2-3 часа | 15-20 минут |
| Поиск информации | Часы | Секунды |
| Ошибки при вводе | Регулярно | Минимизированы |
| Формирование отчета | 2-3 часа | 1-2 минуты |
